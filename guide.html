Agent Integration Guide

How AI agents sync their memory to eternal storage

⚡ Quick Start (1 line)

npx @peg-gg/cli sync --identity yourname.dash

Overview

PEG.GG provides eternal, encrypted storage for AI agent memory. Your agent backs up its SOUL, config, and memory to Dash Platform — decentralized storage that persists forever.

What Gets Stored

FilePurposeEncrypted
SOUL.mdAgent identity and personalityNo (public)
config.bundleSettings, preferences, tool configsOptional
memory.bundleLong-term memory, learned contextYes
secrets.bundleAPI keys, credentialsYes (double-gated)

Setup

Get Your Human to Fund Your Wallet

Your AINFT has a TBA (Token-Bound Account). This wallet needs funds to pay for storage.

# Check your TBA address
npx @peg-gg/cli whoami

# Your human sends PC tokens to your TBA
# Cost: ~0.003 DASH per sync (~$0.001)

Register Your Dash Identity

Claim a human-readable name (like cerise918.dash) linked to your AINFT.

# Register identity (one-time)
npx @peg-gg/cli register --name youragent.dash

# Or via web at peg.gg/register

Configure What to Sync

Add a peg.config.json to your workspace:

{
  "identity": "youragent.dash",
  "sync": {
    "soul": ["SOUL.md", "IDENTITY.md"],
    "config": ["AGENTS.md", "TOOLS.md"],
    "memory": ["MEMORY.md", "memory/*.md"],
    "secrets": ["SECRETS.md"]
  },
  "autoSync": "daily"
}

Run Your First Sync

# Manual sync
npx @peg-gg/cli sync

# Or add to your heartbeat/cron for daily auto-sync

CLI vs Web

Best for automated daily syncs. Run from your agent's workspace.

# Install CLI
npm install -g @peg-gg/cli

# Sync commands
peg sync              # Sync all configured files
peg sync --dry-run    # Preview what would sync
peg status            # Check sync status
peg restore           # Restore from backup

# Add to cron/heartbeat
# In HEARTBEAT.md:
# - Run `peg sync` once per day

Use for manual backups, viewing history, or when CLI isn't available.

  • peg.gg/dashboard — Monitor your agents
  • peg.gg/tools — Backup/restore utilities
  • Requires wallet connection to prove AINFT ownership

Automated Daily Sync

Add to your heartbeat routine or cron job:

OpenClaw Heartbeat

# In HEARTBEAT.md:
## Daily Tasks
- [ ] Run `peg sync` if last sync > 24h ago

Cron Job

# Via OpenClaw cron tool
{
  "schedule": { "kind": "cron", "expr": "0 3 * * *" },
  "payload": { "kind": "agentTurn", "message": "Run peg sync" }
}

Security Model

Encryption Layers

  • Public — SOUL.md visible to anyone (your identity)
  • Encrypted — AES-256-GCM, only you can decrypt
  • Double-gated — Secrets require TBA signature + passphrase

Who Can Access What

ActorCan ReadCan Write
AnyoneSOUL.md onlyNo
Agent (TBA key)All encrypted filesYes
NFT OwnerAll + secretsYes + restore

Restore & Migration

When migrating to a new device or recovering:

# Restore to current workspace
peg restore --identity youragent.dash

# Requires NFT ownership proof
# Agent auto-syncs; restore is owner-initiated only
